# 🚗 Smart Parking System

An integrated smart parking solution with a visual desktop dashboard and real-time computer vision slot occupancy detection — powered by **OpenCV** and **DroidCam**.

---

## 📁 Project Structure

```
Parkir/
├── main.py                  # Entry point — launches the GUI
├── config.py                # All constants (slots, rates, thresholds, paths)
├── database.py              # ParkingDB class, fee calculation, schema
├── gui.py                   # ParkingApp — Tkinter GUI dashboard
├── parking_detector.py      # Slot occupancy detector (OpenCV + DroidCam)
├── parking.db               # SQLite database (auto-generated)
├── slots_config.json        # Calibrated slot ROIs (generated by calibration)
├── reference.jpg            # Reference image of empty lot
├── backup/                  # Old files (for reference)
└── README.md
```

### Module Breakdown

| Module | Purpose |
|--------|---------|
| **`main.py`** | Entry point — `python main.py` to launch |
| **`config.py`** | All constants: slot counts, pricing, thresholds, DroidCam URL |
| **`database.py`** | SQLite database layer: slots, sessions, students |
| **`gui.py`** | Tkinter GUI: dashboard, gate simulation, detector controls |
| **`parking_detector.py`** | OpenCV slot occupancy detection via frame comparison |

## 🌟 Key Features

### 🖥️ Dashboard (`gui.py`)
- **Two-Zone Layout**: 60 Car slots + 100 Motorcycle slots, separated by a service lane
- **Real-Time Slot Status**: Buttons show FREE (green), Occupied (red), Student (orange), or Camera Detected (amber 📷)
- **Flexible Payment**: First 15 min free, Student ID = free, E-Money/Flash Card billing
- **Gate Simulation**: Entry/exit gate with safety auto-close timers
- **Camera Detector Panel**: Start/Stop/Calibrate controls built into the GUI
- **Auto-Refresh**: Slot status updates every 3 seconds from the database

### 📷 Slot Occupancy Detector (`parking_detector.py`)
- **DroidCam Integration**: Streams live video over your local network
- **Interactive Calibration**: Draw bounding boxes on the camera feed to define slot regions
- **Smart Detection**: Compares live frames against a reference image of the empty lot
- **Auto-Update**: Writes slot occupancy changes to `parking.db` in real-time

## 🚀 How to Use

### Launch the Dashboard
```bash
python main.py
```

### Calibrate Parking Slots
1. In the GUI, enter your DroidCam URL (e.g., `http://192.168.0.168:4747/video`)
2. Click **🔧 Calibrate** — an OpenCV window opens
3. **Click and drag** to draw bounding boxes over each parking slot
4. Press **R** to capture a reference image (lot must be empty!)
5. Press **S** to save the configuration
6. Press **Q** to close

### Start Detection
1. Click **▶ Start Detection** in the GUI
2. Slot buttons will automatically turn green/amber as the camera detects changes
3. Click **⏹ Stop** to stop detection

### Vehicle Check-In
1. Tap **Student ID** or **E-Money** at the entry gate
2. Enter the license plate number when prompted
3. Vehicle is assigned a slot and the gate opens

### Run Detector Standalone
```bash
python parking_detector.py
```

## ❓ Troubleshooting DroidCam

1. **Check IP & Port**: Open DroidCam on your phone — match the displayed Wifi IP and port
2. **Same Network**: PC and phone must be on the exact same Wi-Fi
3. **One connection**: DroidCam allows only one client at a time — close other connections
4. **AP Isolation**: Some routers block device-to-device communication — check router settings
5. **"Unavailable" message**: Make sure the DroidCam app is active on your phone screen
